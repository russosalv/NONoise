# SP-2.1 — Port already-generic skills + BMAD integration

## Context

First iteration of **SP-2** (skill library agnostic). SP-2 as a whole is decomposed into
five iterations (SP-2.1..SP-2.5); this spec covers the first one. The objective is to
port the handful of skills that are already largely stack/domain-agnostic from
`reference-project/.claude/skills/`, plus wire an opt-in integration for the BMAD
method skills which are maintained upstream rather than rebundled here.

After SP-2.1 ships, the `create-nonoise` bundle grows from 3 → 5 skills, and Claude Code
users who opt in also get the 5 BMAD agent skills (analyst, architect, tech-writer,
ux-designer, advanced-elicitation) via the upstream `bmad-method` installer.

SP-2.2 and later will tackle stack-bound, methodology, vendor-bound, and domain-bound
skills (see `ROADMAP.md`).

## Goals

- Ship `create-nonoise` v0.2.0 with 2 new agnostic skills in the MVP bundle.
- Add an opt-in BMAD integration that installs the upstream BMAD framework and filters
  it down to 5 agent skills, keeping the output tidy for our target users.
- Preserve the idempotent single-copy skill layout established in SP-1 (skills live
  under `.claude/skills/` only, cross-tool usability documented in each tool's context
  file).
- Verify porting by automated scan: zero `risko|andreani|riscossion|coattiva|dp-*|digital-platform`
  matches in ported skills.

## Non-goals (SP-2.1)

- Porting any other reference-project skill (→ SP-2.2 stack, SP-2.3 methodology,
  SP-2.4 vendor-bound, SP-2.5 domain).
- Rebundling BMAD binaries/sources inside create-nonoise.
- Replacing the MVP skill bundle mechanism with a configurable manifest (still
  hardcoded, tracked under SP-1 decisioni aperte).
- Supporting partial BMAD install via native BMAD flags (we filter post-install).

## Scope summary

**1. Port 2 skills** from `reference-project/.claude/skills/` → `packages/skills/`:
- `playwright-cli` (278 lines + 7 assets)
- `frontend-design` (42 lines)

**2. Add a new CLI prompt step** (`runPrompts`): after the multiselect AI tools,
**always** ask a confirm prompt (regardless of which or how many AI tools were
selected):

> Install BMAD agent skills? (requires network, runs `npx bmad-method install`)

Default `yes`. Exposed also as flags `--bmad` / `--no-bmad`.

Rationale: BMAD skills are markdown files and apply to **all AI tools** (Claude Code,
Copilot, Cursor, Gemini CLI, Codex). Consistent with the SP-1 cross-tool skill decision,
skills live in `.claude/skills/` as the canonical single-copy location — each AI tool's
context file references them via documentation, no duplication on disk. BMAD creates
the `.claude/skills/` folder itself during install if our scaffold didn't. The prompt
is therefore unconditional: whoever is doing AI-assisted development benefits from
having the 5 BMAD agents available.

**3. Scaffold additions**: after the base `scaffold()` pipeline, if `ctx.installBmad`
is `true`:
- Run `npx bmad-method install` as a subprocess with `cwd: projectPath` and
  `stdio: 'inherit'`.
- Call `filterBmadSkills(projectPath)` which removes any `.claude/skills/bmad-*`
  folder not in the allowlist.
- Persist the outcome in `nonoise.config.json`.

**4. MVP bundle update**: `MVP_SKILL_BUNDLE` in `scaffold.ts` grows from 3 → 5 entries:
`graphify-gitignore`, `vscode-config-generator`, `docs-md-generator`, `playwright-cli`,
`frontend-design`. BMAD stays out of the bundle — it's a separate opt-in path.

**5. Type updates**:
```ts
// types.ts — additive
export type ProjectContext = {
  // existing fields...
  installBmad: boolean;
};
```

**6. `nonoise.config.json`** grows:
```json
{
  "installBmad": true,
  "bmadInstalled": true,              // true when the subprocess succeeded
  "bmadInstallError": null            // error message when it failed
}
```

## BMAD integration flow

### Sequence

```
1. runPrompts() → ctx.installBmad (user choice, always asked, default yes)
2. scaffold(ctx):
     a. Existing pipeline (resolve + render + install 5 MVP skills)
     b. If ctx.installBmad:
          - runBmadInstall(projectPath) → { ok: true }     OR
                                          { ok: false, error: '...' }
          - If ok: filterBmadSkills(projectPath) → removed[]
     c. Final write of nonoise.config.json includes BMAD outcome
3. Git init if requested
```

### `runBmadInstall()` contract

```ts
// bmad-runner.ts
export async function runBmadInstall(projectPath: string): Promise<{
  ok: boolean;
  error?: string;
}> {
  try {
    execSync('npx bmad-method install', {
      cwd: projectPath,
      stdio: 'inherit',
      timeout: 300_000,   // 5 min
    });
    return { ok: true };
  } catch (err) {
    return {
      ok: false,
      error: err instanceof Error ? err.message : String(err),
    };
  }
}
```

### `filterBmadSkills()` contract

```ts
// bmad-filter.ts
export const BMAD_ALLOWLIST: ReadonlySet<string> = new Set([
  'bmad-advanced-elicitation',
  'bmad-agent-analyst',
  'bmad-agent-architect',
  'bmad-agent-tech-writer',
  'bmad-agent-ux-designer',
]);

export async function filterBmadSkills(projectPath: string): Promise<string[]> {
  const skillsDir = join(projectPath, '.claude', 'skills');
  // If folder doesn't exist (claude-code not selected), no-op.
  if (!(await exists(skillsDir))) return [];
  const entries = await readdir(skillsDir, { withFileTypes: true });
  const removed: string[] = [];
  for (const e of entries) {
    if (!e.isDirectory()) continue;
    if (!e.name.startsWith('bmad-')) continue;
    if (BMAD_ALLOWLIST.has(e.name)) continue;
    await rm(join(skillsDir, e.name), { recursive: true, force: true });
    removed.push(e.name);
  }
  return removed;
}
```

### Error handling

| Scenario | Behavior |
|----------|----------|
| `npx bmad-method install` not found / network down | Log warning, `installBmad` recorded as requested but `bmadInstalled=false` + `bmadInstallError` populated. Scaffold still succeeds overall. Exit 0. |
| BMAD install prompts interactively | `stdio: 'inherit'` forwards prompts to user terminal; user responds; continuation is automatic. |
| BMAD called without `.claude/` pre-existing (user didn't select Claude Code) | BMAD creates `.claude/skills/` itself during install. `filterBmadSkills` then runs normally on the folder BMAD produced. Non-Claude users consult the skills via their AI tool's context file, which points to `.claude/skills/`. |
| User Ctrl+C during BMAD install | Subprocess exits with non-zero; we catch, record as error, continue scaffold. Exit 0. |
| BMAD install succeeds but writes 0 matching `bmad-*` folders | `removed = []`. No warning (valid outcome if BMAD's scheme changed). |

## Skill porting process

For each of `playwright-cli`, `frontend-design`:

1. Copy entire dir `reference-project/.claude/skills/<skill>/` → `packages/skills/<skill>/`.
2. Scan for domain matches:
   ```bash
   grep -rniE "risko|andreani|riscossion|coattiva|dp-[a-z]+|digital-platform" packages/skills/<skill>/
   ```
3. For each match: remove, generalize, or mark as an example (prefix with "For example…").
4. Check frontmatter: `name` matches folder name, `description` non-empty.
5. Scan for hard-coded paths (e.g., `docs/afu/...`): parameterize or remove.
6. Re-run the scan — must exit with zero matches.
7. Verify skill manifest validator (L4 test) passes: `name` field equals folder name.

Success metric: **zero matches** on the regex above after porting.

## Open decisions (to close during implementation)

- **BMAD install timeout**: 300s is the proposed default. Adjust if users report timeouts
  in real-world network conditions.
- **Interactive vs non-interactive BMAD**: we rely on `stdio: 'inherit'`. If BMAD's
  installer requires TTY and the user is running create-nonoise in a non-TTY
  environment (CI, pipe to file), the install may hang. Mitigation: detect TTY and
  skip BMAD with a warning when stdin is not a TTY.
- **Allowlist drift**: if BMAD renames one of the 5 agent skills in a future version,
  our filter will silently remove it. Add a post-filter check: warn if `removed`
  contained an entry starting with `bmad-agent-` (might indicate we should update
  the allowlist).
