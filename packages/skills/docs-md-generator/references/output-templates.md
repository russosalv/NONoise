# Output templates — canonical skeletons

These are the skeletons the skill emits. Handlebars-style placeholders
(`{{projectName}}`, `{{frameworkVersion}}`) are ONLY meaningful at
scaffolding time (`create-nonoise`). This skill operates on already-
rendered files — it treats those placeholders as ordinary text and
leaves them alone if found. If the skill is creating a file from
scratch (Mode 1 — Bootstrap), fill the placeholders with real values
taken from `AGENTS.md` or asked from the user.

## Skills listing format

Group by purpose. Use bold group headings and inline backticked skill
names, separated by commas. Reference NONoise default grouping:

```
**SDLC orchestrator** — `polly`

**Requirements & discovery** — `requirements-ingest`,
`bmad-agent-analyst`, `bmad-advanced-elicitation`

**Architecture & validation** — `arch-brainstorm`, `arch-decision`,
`quint-fpf`, `bmad-agent-architect`

**Sprint & implementation** — `sprint-manifest`, `atr`

**Brownfield** — `reverse-engineering`

**Ops & observability** — `ops-skill-builder`, `observability-debug`

**Integrations** — `spec-to-workitem`, `playwright-cli`,
`frontend-design`, `bmad-agent-ux-designer`, `bmad-agent-tech-writer`

**Generators** — `vscode-config-generator`, `docs-md-generator`,
`design-md-generator`

**Utility** — `skill-finder`
```

If the project installed vendored superpowers, add a `**Superpowers**`
group at the end with the full list (or reference the
`.claude/skills/superpowers/` folder).

## Language notes

- Detect the language of the source `AGENTS.md` by looking at the
  intro prose (not H2 headings, which are often English even in
  Italian docs).
- If mixed, default to English and flag it in the summary.
- Never translate managed-block bodies — they are owned by their
  skills and preserved byte-for-byte.
